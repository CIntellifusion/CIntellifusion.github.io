## 第一节 内存和指针

### 空指针 野指针 

**C语言的层面** 声明指针之后需要自动初始化，初始化的方式是

```c
#define type char
type* pointer = NULL;
```

任何类型的指针都应该直接赋值或者初始化为NULL。

**不初始化的指针是一个非0整数，其指向的内存1. 可能没有意义 2. 可能不在数据段** 如果不对这个指针进行引用，则不会出问题，因为不引用一个指针，他就是一个整数。



> NULL是什么：NULL是由宏定义的无类型指针，其值是 0。
>
> 引用是什么: 引用是从一个用整数值标记的地址中读入数据



**初始化一个指针**作用是在引用之前可以判断是否可以引用这个指针。

在初始化良好的程序里，如下代码可以避免引用无效的指针

```c
/*pointer 是一个指针，在程序的其他地方定义*/
if(pointer){/*a set of instructions*/}
else {printf("cite a null pointer\n");}
```

如果指针没有初始化，它会跳入第一个分支，并且对这个分支产生引用，然后引发错误



#### 野指针

野指针是出了空指针外，指向无效内存的指针。 通常包括未经初始化的指针，和指向原本存在值，但是后来被free掉的指针。

> 为什么空指针不是危害，而野指针是：因为我们总可以在引用一个指针之前区别空指针和有效指针

#### 引用相同指针

```c
void swap(int * x, int *y)
{
*x = *y ^*x;
*y = *y ^*x;
*x = *y ^ *x;
}
```



如上程序在正常情况下，会将x,y的值互换一下，而不用借助一个空瓶子。

> 呜呜 2018年的记忆突然开始攻击我， 刚刚接触C++的时候，就写了如何交换两个变量值的例子。当时老师打了一个比方，你现在有一瓶酱油和一瓶醋，你要把酱油装进醋瓶子，然后把醋装进酱油瓶子，你应该怎么办？ —————— 你是不是应该找一个空瓶子来。

> 当你了解位运算之后，你应该发现， 并不用，你只需要将酱油瓶子和醋瓶子进行三次异或运算就可以了。



言归正传，上述程序有一个小bug,在x,y是相同的指针的时候， 会把这个指针的值设置为0。注意，我们此时讨论的是指针指向相同内存，也就是你只有一个瓶子的时候（大雾，此时你把一瓶酱油和它自己异或一下，这瓶酱油就没了）。



由此，我们引出了一类非常常见的指针的bug: 将两个相同的指针进行运算。 

```c
void twiddel1(int *xp ,int * yp )
{
*xp += *yp;
*yp += *xp;
}

void twiddle2(int *xp,int *yp)
{
*xp+=2**yp;
}
```

在一般情况下，两个程序产生相同的结果，但是当xp和yp是同一个地址时，他们会产生不同的结果。这叫**内存别名引用**

**所以，希望在引用指针运算时，一定要考虑指针是否指向同一个位置**

### 多重指针

多重指针在概念上和一重指针同样好理解。其相同之处在于，他们指向的都是一个内存。区别在于内存里装的东西不一样。 

But it is always easier said than done. 

下面举一个二重指针的例子

```c
void cut(char **src)
{
    /*this function aims to cut the string in 2 half,
    and throw away the former half
    */
    assert(src!=NULL);
    int len = strlen(*src);
    assert(len>2);
    len = len/2;
    *src += 2;
}
int main()
{
    char * string = (char*)malloc(20);
    scanf("%s",string);// 这里
    cut(&string);
}
```

> 我深知我是解释不清楚的，所以如果存在读者的话，去跑一遍代码就明白了

此处由于是一个函数调用，函数会把src的地址放入%rdi寄存器，然后把%rdi的内存里面的值修改。

如果是传入string的地址的话，会把string的地址放进%rdi,然后将寄存器里面的值改掉，然后返回。因此就不涉及内存，所以原本的string的地址没有变化。 

用指针也可以理解，但是涉及作用域，比较繁琐。

### 寻址模式和mov

寻址模式指的是确定内存中的一个地址的方法。

以最典型的寻址方式为例

```assembly
0x2(%rdi,%rsi,2)
```

该代码表示的地址是%rdi+2*%rsi+0x2

%rdi是基地址，%rsi和0x2分别是偏移量

这种寻址模式常见在数组的寻址，对数组首地址进行偏移



### load effective address

```
movl  0x2(%rdi,%rsi,2),%eax
lea   0x2(%rdi,%rsi,2),%rbx
```

此节可以帮助深入理解 整数和地址 是在什么程度上等价的

上面两句的区别在于，%rax的值是一个32位的整数，%rbx是64位的地址。

lea是用来加载有效的地址的，但是并不访问内存。

在C语言中，eax 和 rbx 的对应关系可以写成

```c
eax ==*(rbx)
```



但是lea指令也可以用来计算一些不是地址的整数，比如

```
movl 0x2,%rdi
movl 0x2,%rsi
lea   0x2(%rdi,%rsi,2),%rbx
```

此时rbx的值就是2+2*2+2=8

但是由于并不访问内存或者说间接引用这个指针，所以并不会报错。



编译器常用lea来优化常数乘法，lea不属于算数运算符，有专门的硬件实现寻址操作，所以其做乘法的周期快于算数乘法。



从上面的例子可以看出，在汇编层面，整数和指针并无区别，lea指令可以用在非地址的整数上。

### 数据结构

数据结构可以是程序中所有数据的总称。 int,float等等基础的数据类型也可以被认为是数据结构，但是他们是不可分的最基础的数据结构，类似原子在化学中的地位。 基础的数据结构可以组成复杂的数据结构，从而数据的表示能力得到了增强。 数据结构加上对数据结构的方法，就成为了一个完整的对象。（挖坑：群是一个集合+一个运算。 群和对象似乎有些相通之处，等我学了抽代回来)

#### 同质的数据结构——数组

数组是若干相同的数据类型组成的拥有连续空间的数据结构。

```c
typedef struct list
{
int a;
struct list * next;
}list;
 list L[20];
```

上面这段代码中的L数组，显然是一个同质的数据结构，他的每一个元素都是list类型。

注意：Python中的list类型实际上不是数组，因为它允许异质的数据在同一个list里面。 另外，list也有自己的方法，所以它其实是一个对象类型。 python中的一切都是对象。

 但是对于list来说，它却是一个异质的数据结构。

对List这个结构体来说，它包括两个成员，一个整型和一个指针，所以是异质的数据类型。

但是在这个结构体被分配内存的时候，int和list*是紧挨着分配的。这样极大方便了寻址，同时增强了数据结构表达能力。

## 第四节 内存管理和指针

### malloc函数和calloc函数

malloc函数负责在heap部分分配一部分内存，然后返回分配的地址的首地址，类型是无类型指针。

malloc函数不会对分配的空间进行初始化。

如果分配失败会返回一个NULL指针。

heap是一块非常大的内存区域，在我自己的电脑上测试出可以分配128752141byte内存，大概是100M左右。

和栈区自动回收不同，堆区分配的内存只能用free来释放，如果不释放就会一直保持占用状态。

### 内存泄漏和指针

对于每一块分配了的内存，要想寻找他们，就要通过指针。

前面提到malloc的内存必须要通过free来释放，所以如果我们丢失了一块内存的位置，那么就无法释放这块内存。

```c
char* head = (char*)malloc(20);
// free(head);
head = NULL:
```

由于head指向了一个新的地方，而heap里面分配的内存没有释放，所以我们在程序进行过程中，再也不能使用这块内存。

如果程序需要反复分配内存，不及时free可能会导致malloc失败。

没有释放的内存在程序结束之后由操作系统释放。

### 寄存器指针rbp rsp 和 rip

rbp指向当前活跃函数的起点，

rsp指向数据栈顶

rip指向下一条将要执行的指令，也叫程序计数器

**在正常情况下，rsp和rip指向的范围是互不相交的**，所以可以理解成两个独立的栈：数据栈和程序栈

栈溢出攻击的最核心的思路在于操控rsp和rip指针到我们想要的地方



#### 修改函数返回地址的原理是什么

或许小标题还不够清楚。

函数在调用的时候，总是需要返回主函数的，所以需要将返回地址压入栈中，在函数执行完毕之后，弹出这个地址，然后将程序返回到主函数中。

栈溢出攻击的第一步就是阻止函数返回到原来的地址，同时返回到攻击者希望的地址。 所以小标题提出的问题是： 程序在我们修改了地址的具体值之后，是如何将修改后值识别为一个地址的。

| data stack                   |       | instrctions |       |
| ---------------------------- | ----- | ----------- | ----- |
| return address of a function | <-rsp |             |       |
|                              |       | c3          | <-rip |
|                              |       |             |       |

表格中所示是一个函数执行完成后准备返回时的状态，rsp指向的栈顶恰好是调用函数时压入栈中的返回地址，c3是ret的机器代码，rip指向的恰好是ret指令。

ret指令分为两步，第一步pop返回地址，第二步将rip修改为得到的值。

c3指令默认此时栈顶元素是函数应当返回的地址，所以弹出后跳转。 通过缓冲区溢出，将这个地方原来的值修改了之后，rip就会指向攻击者期望的地方，

#### 注入代码的原理是什么
